## ðŸ“– Documentation Highlights

### Technical Architecture

**Key Design Decision:** Client-side anonymous sessions with JWT tokens

**Why this approach?**
- âœ… No database writes (zero cost)
- âœ… Instant scaling (client-side)
- âœ… Auto-cleanup (30-minute expiry)
- âœ… Privacy-friendly (no data stored)

### Security Model

**5 layers of security:**
1. JWT token signing with NEXTAUTH_SECRET
2. Short 30-minute expiry windows
3. Rate limiting (same as authenticated users)
4. Input sanitization (DOMPurify)
5. No PII collection or tracking

### User Experience

**Smooth upgrade flow:**
- Guest hits limit â†’ Upgrade modal

### JWT Session Structure

```typescript
interface GuestSessionToken {
  sub: string;           // Subject (guestId)
  iat: number;           // Issued at
  exp: number;           // Expiry (30 min)
  isGuest: true;         // Guest flag
  guestId: string;       // UUID
  sessionType: 'guest';  // Type
}
```
